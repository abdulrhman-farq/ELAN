# ابدئي من هنا — دليل تشغيل ELAN (إيلان)

هذا دليل مبسّط بالعربية لتشغيل النظام وتشغيله اليومي، وكيف تجهّزينه للإطلاق الحقيقي.

> **ملاحظة:** الحسابات التجريبية وقاعدة البيانات جاهزة. لا تحتاجين أي مفاتيح سرية
> لتجربة النظام الآن.

---

## ١) كيف تشغّلينه (٣ خطوات)

افتحي «الطرفية» (Terminal) داخل مجلد `elan` ثم نفّذي:

```bash
npm install      # تنزيل المكوّنات (مرة واحدة)
npm run setup    # تجهيز الإعدادات تلقائياً
npm run dev      # تشغيل التطبيق
```

بعدها افتحي المتصفح على: **http://localhost:3000**

### حسابات الدخول التجريبية

| الدور | البريد | كلمة المرور |
| ----- | ------ | ----------- |
| عضوة  | `noor@elan.demo`  | `elan1234` |
| مسؤولة (المالكة) | `owner@elan.demo` | `elan1234` |

- صفحة الدخول فيها زرّان سريعان: «دخول كعضوة تجريبية» و«دخول كمسؤولة».
- لوحة الإدارة على الرابط: **http://localhost:3000/admin** (تظهر للمسؤولة فقط).

---

## ٢) ماذا تستطيعين تجربته الآن

**كعضوة:**
- تصفّح **الجدول**: الحصص لكل يوم مع حالة كل حصة (متاح / قائمة انتظار / مكتمل / الحجز مغلق / محجوز).
- افتحي تفاصيل حصة ثم **احجزي**، أو **انضمي لقائمة الانتظار** إذا كانت ممتلئة.
- **ألغي حجزاً** من شاشة «حجوزاتي» (مع احترام نافذة الإلغاء ١٢ ساعة).
- **اشتري** عضوية أو باقة حصص من شاشة «العضويات» (دفع وهمي فوري — يضيف الرصيد فعلاً).
- بدّلي اللغة (عربي/English) من «حسابي».

**كمسؤولة:** لوحة تعرض حصص اليوم ونسبة الإشغال وإيراد اليوم.

---

## ٣) كيف يعمل المنطق المهم (باختصار)

- **نافذة الحجز:** يفتح الحجز قبل بداية الحصة بـ٣ أيام ويُغلق عند بدايتها.
- **الأهلية:** النظام يتحقق من المستوى (١ ← ١.٥ ← ٢) ومن الرصيد قبل السماح بالحجز،
  ويُظهر سبب المنع على الزر (مثلاً «المستوى غير مناسب» أو «لا يوجد رصيد»).
- **الخصم:** يُحسب من **العضوية** أولاً، وإلا من **رصيد الحصص**.
- **قائمة الانتظار:** عند إلغاء عضوة مؤكَّدة، تُرقّى أول عضوة في الانتظار تلقائياً
  ويُرسَل لها إشعار (إن كانت موافِقة على القناة).
- **الإلغاء المتأخر / عدم الحضور:** داخل نافذة الـ١٢ ساعة يُفقَد الرصيد وتُسجَّل غرامة.

---

## ٤) المهام اليومية (من غير برمجة)

> لوحة الإدارة حالياً للعرض فقط. الإجراءات التشغيلية (تحضير، عدم حضور، إضافة حصة،
> استرجاع رصيد) **جاهزة في قاعدة البيانات** وستُضاف لها أزرار في اللوحة لاحقاً.
> حتى ذلك الحين يمكن تنفيذها من لوحة Supabase. اطلبي المساعدة عند الحاجة.

---

## ٥) عندما تجهزين للإطلاق الحقيقي — المفاتيح المطلوبة

أضيفي القيم في ملف `.env.local` (انسخي الأسماء من `.env.example`):

| الخدمة | المفاتيح | الغرض |
| ------ | -------- | ----- |
| Supabase | `NEXT_PUBLIC_SUPABASE_URL`، `NEXT_PUBLIC_SUPABASE_ANON_KEY` | قاعدة البيانات والدخول (استخدمي مشروعك الخاص للإطلاق) |
| الدخول بالجوال | تفعيل مزوّد SMS داخل Supabase | لتفعيل الدخول برمز OTP بدل البريد |
| المدفوعات (ميسر) | `MOYASAR_PUBLISHABLE_KEY`، `MOYASAR_SECRET_KEY` | مدى/بطاقة/Apple Pay الحقيقي |
| واتساب | `WHATSAPP_ACCESS_TOKEN`، `WHATSAPP_PHONE_NUMBER_ID` | إشعارات الحجز والترقية الحقيقية |
| الأخطاء (اختياري) | `SENTRY_DSN` | تتبّع الأخطاء |

> بمجرّد إضافة مفاتيح أي خدمة، يتحوّل النظام تلقائياً من «الوضع الوهمي» إلى المزوّد
> الحقيقي. بدون مفاتيح، كل شيء يعمل ببيانات تجريبية وإشعارات تُسجَّل في السجل.

---

## ٦) نصائح

- خذي **نسخة احتياطية يومية** لقاعدة البيانات من لوحة Supabase.
- افصلي بيئة **التجربة** عن بيئة **الإنتاج** (مشروعَا Supabase منفصلان).
- لا تُخزَّن بيانات البطاقات لدينا إطلاقاً — فقط عبر ميسر.

لمزيد من التفاصيل التقنية، انظري ملف `DECISIONS.md`.
