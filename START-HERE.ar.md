# ابدئي من هنا — دليل تشغيل ELAN (إيلان)

هذا دليل مبسّط بالعربية لتشغيل النظام وتشغيله اليومي، وكيف تجهّزينه للإطلاق الحقيقي.

---

## ١) كيف تشغّلينه (٣ خطوات)

افتحي «الطرفية» (Terminal) داخل مجلد `elan` ثم نفّذي:

```bash
npm install      # تنزيل المكوّنات (مرة واحدة)
npm run setup    # تجهيز الإعدادات تلقائياً
npm run dev      # تشغيل التطبيق
```

بعدها افتحي المتصفح على: **http://localhost:3000**

### وضع التجربة المحلي (اختياري)

لتفعيل البيانات التجريبية والدخول السريع **محلياً فقط**، أضيفي في `.env.local`:

```bash
NEXT_PUBLIC_ELAN_DEMO=true
```

ثم أعدي تشغيل `npm run dev`. ستظهر أزرار «دخول كعضوة تجريبية» في صفحة الدخول.

| الدور | البريد | كلمة المرور |
| ----- | ------ | ----------- |
| عضوة  | `noor@elan.demo`  | `elan1234` |
| مسؤولة (المالكة) | `owner@elan.demo` | `elan1234` |

> **في الإنتاج** (`elan-rosy.vercel.app`) وضع DEMO **مُعطّل دائماً** — يجب تسجيل الدخول بحساب حقيقي.

---

## ٢) ماذا تستطيعين تجربته الآن

**كعضوة:**
- تصفّح **الجدول** والحجز وقائمة الانتظار والإلغاء.
- **طلب شراء** عضوية أو باقة من «العضويات» (يُنشئ طلباً بانتظار تأكيد الدفع من الاستوديو).
- بدّلي اللغة (عربي/English) من «حسابي».
- اطلعي على **سياسة الخصوصية** و**الشروط** من صفحة الدخول أو «حسابي».

**كمسؤولة:** لوحة إدارة كاملة — جدول، عضوات، مدفوعات، تقارير، إعدادات الاستوديو، وفحص النظام (`/admin/health`).

---

## ٣) كيف يعمل المنطق المهم (باختصار)

- **نافذة الحجز:** تُضبط من إعدادات الاستوديو (افتراضياً ٧ أيام قبل الحصة).
- **الإلغاء المجاني:** خلال المدة المحددة في الإعدادات (افتراضياً ١٢ ساعة قبل الحصة).
- **الشراء:** يُنشئ دفعة «معلّقة» — الرصيد يُضاف فقط بعد تأكيد الدفع (يدوياً أو عبر بوابة الدفع).
- **قائمة الانتظار:** عند إلغاء حجز، تُرقّى أول عضوة في الانتظار تلقائياً.

---

## ٤) المهام اليومية

معظم العمليات التشغيلية متاحة من لوحة الإدارة: تحضير، عدم حضور، بيع باقات، تأكيد مدفوعات، استرجاع، وجدولة الحصص.

---

## ٥) عندما تجهزين للإطلاق الحقيقي — المفاتيح المطلوبة

أضيفي القيم في Vercel (Production) أو `.env.local` (انسخي الأسماء من `.env.example`):

| الخدمة | المفاتيح | الغرض |
| ------ | -------- | ----- |
| Supabase | `NEXT_PUBLIC_SUPABASE_URL`، `NEXT_PUBLIC_SUPABASE_ANON_KEY` | قاعدة البيانات والدخول |
| الموقع | `NEXT_PUBLIC_SITE_URL` | روابط الدخول والبريد |
| المدفوعات (ميسر) | `MOYASAR_SECRET_KEY` | مدى/بطاقة — يُفعَّل تلقائياً عند الإضافة |
| البريد (Resend) | `RESEND_API_KEY`، `RESEND_FROM` | إشعارات البريد |
| واتساب | `WHATSAPP_ACCESS_TOKEN`، `WHATSAPP_PHONE_NUMBER_ID` | إشعارات واتساب |
| الأخطاء | `SENTRY_DSN` | تتبّع الأخطاء |
| Webhook | `PAYMENT_WEBHOOK_SECRET`، `SUPABASE_SERVICE_ROLE_KEY` | تأكيد الدفع الآلي |

### بيئة الاختبار (Staging)

راجعي `docs/STAGING_SETUP.md` — مشروع Supabase منفصل + متغيرات Vercel Preview.

### فحص الجاهزية

- **محلياً:** `npm run typecheck && npm run lint && npm test && npm run build`
- **E2E:** `npm run test:e2e`
- **بعد النشر:** `/admin/health` في لوحة الإدارة

---

## ٦) نصائح

- خذي **نسخة احتياطية يومية** لقاعدة البيانات من لوحة Supabase.
- افصلي بيئة **staging** عن **production**.
- لا تُخزَّن بيانات البطاقات لدينا — فقط عبر ميسر.
- طبّقي ترحيل `0024_launch_readiness.sql` على قاعدة الإنتاج إن لم يُطبَّق بعد.

لمزيد من التفاصيل التقنية، انظري `docs/LAUNCH_ACCESS_CHECKLIST.md` و`DECISIONS.md`.
