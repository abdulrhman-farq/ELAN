# ÉLAN — دليل التدفّقات (Flows)

نظام حجز استوديو بيلاتس نسائي (الرياض) + لوحة إدارة + محاسبة مالية.
كل المبالغ تُخزَّن بالهللات (عدد صحيح؛ 1 ر.س = 100 هللة).

---

## 0) الأدوار والوصول

| الدور | الدخول | ماذا يرى |
|------|--------|----------|
| **المديرة (Admin)** | بريد + كلمة مرور | لوحة الإدارة `/admin` كاملة |
| **المشتركة (Member)** | بريد + رابط سحري (Magic link) | تطبيق العضوة `/` (جدول، حجز، حجوزاتي، بروفايل) |
| **زائر غير مسجّل** | — | تطبيق العضوة بوضع **تجريبي** (عرض فقط) |

الحماية: كل عمليات الإدارة محميّة بـ`is_admin()` (RLS + إعادة فحص في الـactions). كل بيانات المشتركة محميّة بـ`current_member_id()` (تقرأ سجلّها فقط).

---

## 1) تدفّق دخول المديرة

1. `/login` → تُدخل البريد + كلمة المرور → «دخول».
2. `signInWithPassword` ثم فحص `is_admin()`.
3. إذا مديرة → تحويل إلى `/admin`، وإلا إلى `/`.
4. أي محاولة دخول لـ`/admin` بدون جلسة مديرة → تحويل تلقائي لـ`/login`.

---

## 2) تدفّق دخول المشتركة (رابط بريد)

1. المديرة تنشئ العميلة **ببريدها** (انظر تدفّق 4).
2. المشتركة تفتح `/login` → «دخول العميلة برابط عبر البريد» → تُدخل بريدها.
3. `signInWithOtp` يرسل رابطاً إلى بريدها.
4. تفتح الرابط → `/auth/callback`:
   - يبادل الكود بجلسة.
   - **يربط** `members.auth_user_id = auth.uid()` بمطابقة البريد (مرة واحدة).
5. تحويل إلى `/` → التطبيق يحلّ سجلّها الحقيقي ويعرض **اسمها + رصيدها**.

> شرط التفعيل في Supabase: Email provider مفعّل + `…/auth/callback` ضمن Redirect URLs + إعداد SMTP لإرسال الروابط.

---

## 3) تدفّق لوحة التحكم (Dashboard) — `/admin`

عند الفتح تُحسب من بيانات حيّة:
- **مؤشرات اليوم**: حجوزات اليوم، نسبة الإشغال، عضوات جدد هذا الأسبوع، إيراد الشهر.
- **تنبيهات الحصص**: حصص مكتملة / «باقي مقعد واحد» + عدّاد حجوزات جديدة اليوم.
- **مهام متابعة مستحقة**: المهام المفتوحة المتأخرة مع رابط العميلة.
- **حصص اليوم**: الوقت، الحصة، المدرّبة، الحجز x/y، الحالة (مفتوحة/باقي مقعد/مكتملة).
- **قائمة الانتظار** اليوم + **الأعلى إشغالاً**.

---

## 4) تدفّق إنشاء عميلة + اختيار باقة (للمديرة فقط)

`/admin/members` → «+ عميلة جديدة»:
1. تعبّي: الاسم (مطلوب)، الجوال، البريد، المصدر، الحالة (مهتمة/تجريبية/نشطة/منقطعة).
2. تختار **الباقة**: بدون / حصة مفردة / 8 حصص / 12 حصة.
3. «حفظ»:
   - `createMemberAction` ينشئ سجل العميلة.
   - إذا فيه باقة → يستدعي `sellPackageAction` تلقائياً: يضيف الرصيد + يسجّل عملية دفع (تُحتسب في الإيراد) + يكتب سجل تدقيق.

---

## 5) تدفّق ملف العميلة — `/admin/members/[id]`

- **تعديل البيانات**: الاسم/الجوال/البريد/المصدر/الحالة.
- **واتساب سريع**: 4 رسائل جاهزة (متابعة، تذكير تجريبية، تذكير تجديد، إعادة تفعيل) — تفتح wa.me برقم سعودي مُطبّع.
- **مهام المتابعة**: إضافة مهمة (عنوان + استحقاق)، إكمالها بنقرة، تمييز المتأخرة.
- **الملخص المالي**: إجمالي المدفوع، إجمالي الخصومات، قيمة الحصص المحضورة، قيمة الرصيد المتبقي، قيمة عدم الحضور، قيمة الحصص المجانية.
- **بيع باقة/حصص**: اختيار الباقة + السعر الصافي + خصم (نسبة/ثابت/كود) → بيع يُحتسب في الإيراد.
- **سجل المتابعة**: ملاحظات.
- **آخر الحجوزات**: لكل حجز أزرار **خصم** و**مجانية (Comp)**.

---

## 6) تدفّق المحاسبة المالية (قلب النظام)

السعر المرجعي (من دراسة الجدوى): **الحصة الصافي 150 ر.س، ضريبة 15%، الإجمالي 172.5 ر.س**.

محرّك التسعير `computePrice` (هللات):
```
الصافي قبل الخصم
 → الخصم (نسبة على الصافي / مبلغ ثابت)  [الصافي لا ينزل تحت 0]
 → الصافي بعد الخصم
 → الضريبة = الصافي بعد الخصم × 15%
 → الإجمالي = الصافي بعد الخصم + الضريبة
```

مصدر التسعير لكل حجز (`pricing_source`):
- **single**: بيع نقدي مفرد → يُنشئ دفعة + يُحتسب إيراد.
- **package_credit**: خصم حصة من الرصيد، تُسجَّل قيمة الحصة (list value)، بدون نقد جديد.
- **unlimited_membership**: بدون خصم رصيد، تُسجَّل قيمة الاستهلاك للتقارير فقط.
- **complimentary**: حصة مجانية → الإجمالي 0 مع حفظ القيمة المرجعية.
- **manual**: تعديل يدوي.

كل تغيير سعر/خصم/مجانية يُكتب في **سجل التدقيق** (`pricing_audit`): مَن، متى، السبب، القيمة القديمة، الجديدة.

---

## 7) تدفّق أكواد الخصم — `/admin/promo`

1. إنشاء كود: الكود، النوع (نسبة/ثابت)، القيمة، يبدأ/ينتهي، حد الاستخدام، حد لكل عميلة.
2. عند تطبيق كود في بيع → `resolvePromo` يتحقق: مفعّل؟ ضمن الصلاحية؟ لم يتجاوز الحدود؟
3. يُسجّل الاستخدام في `promo_redemptions` (يربط بالعميلة/الحجز/الدفع).
4. تفعيل/إيقاف أي كود بنقرة.

---

## 8) تدفّق التقارير — `/admin/reports` (آخر 30 يوم)

تفصل بوضوح:
- **المبيعات الإجمالية** (شامل الضريبة) · **الصافية** · **الضريبة المحصّلة** · **الخصومات**.
- **استهلاك الباقات** · **استهلاك العضويات غير المحدودة** · **قيمة الحصص المجانية** · **قيمة عدم الحضور المفقودة**.
- الإيراد حسب النوع (عضويات/باقات/حصص مفردة/جلسات خاصة/غرامات).
- الحجوزات حسب الحالة + قيمة الإلغاءات.

---

## 9) تدفّقات إدارية أخرى

- **الجدول** `/admin/schedule`: حصص الأسبوعين القادمين مجمّعة بالأيام (عرض حقيقي).
- **المدرّبات** `/admin/trainers`: المدرّبات + عدد حصص كل وحدة هذا الأسبوع.
- **تصدير CSV** `/admin/members/export`: الاسم، الجوال، البريد، الحالة، المصدر، حالة العضوية، الرصيد، تاريخ التسجيل (UTF-8 لإكسل). محمي — يرفض غير المديرة.
- **الإعدادات** `/admin/settings`: معلومات الاستوديو وسياسات الحجز (إعداد).

---

## 10) تدفّق المشتركة (تطبيق العضوة، بعد تسجيل الدخول)

1. **الرئيسية** `/`: تحية باسمها + رصيد الحصص + الحصة القادمة.
2. **الجدول** `/schedule`: الحصص الحقيقية القادمة + حالتها (متاحة/قائمة انتظار/مكتملة).
3. **تفاصيل الحصة** `/class/[id]`: الأهلية + زر الحجز.
4. **الحجز** → `book_class_self`:
   - يتحقق من الأهلية + الرصيد + السعة (6) + نافذة الحجز.
   - يخصم حصة، يؤكّد الحجز (أو يضيفها لقائمة الانتظار إن كانت مكتملة).
5. **حجوزاتي** `/bookings`: حجوزاتها الحقيقية + إلغاء (`cancel_booking_self`).
6. أي حجز جديد يظهر فوراً في **تنبيهات لوحة الإدارة** وفي إشغال الحصة.

---

## 11) السعة والتنبيهات

- سعة كل حصة = **6 مقاعد** (عدد أجهزة الريفورمر).
- تنبيهات تلقائية في لوحة الإدارة: **اكتملت** / **باقي مقعد واحد** / **حجوزات جديدة اليوم**.

---

## 12) نموذج البيانات (مختصر)

- `members` (+ `auth_user_id`, `lead_status`, `source`) · `member_notes` · `member_tasks`
- `class_types` (+ `base_net_halalas`, `vat_bps`, `default_capacity=6`) · `class_instances` · `instructors`
- `bookings` (+ حقول المحاسبة: base_net/discount/final_net/vat/final_gross/pricing_source/list_value… بالهللات)

- `credit_ledger` (حركة الرصيد) · `membership_plans` · `credit_packs`
- `promo_codes` · `promo_redemptions` · `pricing_audit`

---

## 13) إعداد مطلوب من Supabase (لتشغيل دخول المشتركات)

1. Authentication → Providers → **Email**: مفعّل.
2. Authentication → URL Configuration → **Redirect URLs**: أضيفي `https://elan-rosy.vercel.app/auth/callback`.
3. **SMTP** (Auth → SMTP): مزوّد بريد لإرسال روابط الدخول.

---

## 14) دورة كاملة (مثال واقعي)

1. المديرة تنشئ «سارة» ببريدها وتختار باقة 8 حصص → الرصيد 8، الإيراد +قيمة الباقة، سجل تدقيق.
2. سارة تدخل بالرابط → ترى اسمها ورصيدها (8).
3. تحجز «Reformer Flow» → الرصيد 7، الحجز مؤكّد، تنبيه في لوحة الإدارة.
4. تتغيّب → المديرة تعلّمها «لم تحضر» → تظهر القيمة في «قيمة عدم الحضور» بالتقارير.
5. المديرة ترسل لها رسالة واتساب «إعادة تفعيل» وتضيف مهمة متابعة باستحقاق.
